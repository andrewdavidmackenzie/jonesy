## Top Optimization Targets

1. **`symbol_index_new`** - 84.5 ms (8.4%)
   - Iterates all symbols in binary
   - Demangles every symbol name eagerly
   - Allocates String for each demangled name
   - Then sorts entire collection

2. **`call_graph_build`** - 72.6 ms (7.2%)
   - Disassembles entire text section
   - Builds caller/callee relationships

3. **`prune_call_tree` + `build_call_tree_sequential`** - 52 ms (5.1%)
   - Tree traversal and pruning operations
